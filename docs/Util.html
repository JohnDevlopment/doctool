NAME

Doctool::Util - Utility subroutines for Doctool

VERSION

Version 0.10

SYNOPSIS

This module exports a number of subroutines that help with the functionality of Doctool. Strictly speaking, these subroutines are not only useful for Doctool; they can be used in general practice.

EXPORT

The following subroutines are exported by this module.

append( variable, value )

Appends value to the end of variable and returns a reference to it. Accepts an array or a scalar.

echo( arg, ... )

Prints out one or more arguments using the print function. The difference is that echo automatically appends a newline character

funcref( funcname )

Returns a reference to the function funcname if it exists, otherwise it returns undef.

Use this function to test for the existtence of a particular function.

max( a, b )

Returns the greater of the two parameters.

min( a, b )

Returns the lesser of the two parameters.

printarray( array )
printarray( arrayref )

Prints the contents of an array. The parameter can be any valid array or array reference.

printhash( hash )
printhash( hashref )

Prints the contents of a hash. The parameter can be any valid hash or hash reference.

str( arg, ... )

Concatenates the arguments into a string. Requires at least one argument.

strextract( PATTERN, STRING )
strextract( PATTERN, STRING, GROUPLIST )

Extracts a substring from STRING according to PATTERN, which is expected to be a precompiled regular expression. Use the qr() function (documented in "qr/STRING/" in perlfunc) to make PATTERN.

In the first form, strextract() extracts the portion of STRING that matches PATTERN, or an empty string in the case of failure. If it is called in a scalar context, only the first match is returned. But in a list context a list of all matching substrings are returned.

In the second form, strextract() returns one or matches of subexpressions in PATTERN. In a scalar context only the first match is returned. In a list context all matches are returned as an array, and each element corresponds to a group number parameter. For example, strextract(qr/(like).*(donuts)/, $string, 1, 2) will return a list of strings that correspond to $1 and $2, respectively.

In all examples, assume the following:

    my $string = "I like donuts";

Example 1 (returning a whole match):

    my $match = strextract(qr/donuts/, $string);
    echo $match; # prints "donuts"

Example 2 (returning a subgroup of a match):

    my $match = strextract(qr/do(nuts)/, $string, 1);
    echo $string, "I like", $match; # prints "I like donuts! I like nuts"

Example 3 (getting multiple subexpression matches):

    my @matches = strextract(qr/(like).*(donuts)/, $string, 1, 2);
    echo @matches; # prints "like donuts"
